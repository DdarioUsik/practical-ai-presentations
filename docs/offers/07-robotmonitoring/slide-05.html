Технические особенности

Вся логика реализована в n8n с использованием узлов Schedule Trigger, HTTP Request, Code, PostgreSQL и Email. Кастомный JavaScript выполняет расчёт коэффициентов утилизации по формулам OEE (Overall Equipment Effectiveness). Базой данных выступила система Airtable со встроенным дашбордом. Оссобенностью являет дизайн системы выявления аномалий: настроены пороговые значения для каждого типа робота с учётом специфики производственного цикла. При снижении уровня перформанса (норма - average performance > 80%) более 3 дней подряд, система автоматически инициирует эскалацию внутренней команде. Система обрабатывает данные с парка из 170+ роботов (~850 ежедневных задач) с интервалом опроса 24 часа, пакетами. Исторические данные хранятся для построения трендов и прогнозирования необходимости технического обслуживания. ___ Технология: n8n workflow automation Время внедрения: 60 рабочих дней
