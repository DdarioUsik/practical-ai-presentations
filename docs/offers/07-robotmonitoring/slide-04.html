Ключевые компоненты workflow

  • Scheduled триггер. запускает сбор данных по расписанию для регулярного обновления метрик.
  • HTTP Request модуль. получает телеметрию с API контроллеров роботов (статус, время работы, простои).
  • Модуль агрегации данных. рассчитывает коэффициенты производительности и KPI по каждому роботу и линии.
  • Интеграция с базой данных Airtable. сохраняет исторические данные для анализа трендов и построения отчётов.
  • Модуль расчёта аномалий. выявляет отклонения от нормальных показателей утилизации.
  • Dashboard обновление Airtable. передаёт актуальные данные в систему визуализации для мониторинга и отчетов.​
  • Alert-уведомления. отправляет оповещения по email при критическом снижении утилизации или сбоях, а также отправляет оформленные письма для клиентов .
