## Обязательные вопросы стратсессии

### 1) Цели и фокус
- [ ] Какой один результат бизнес должен получить через 90 дней?
- [ ] Что приоритетнее: операционная управляемость или рост продаж?
- [ ] Какой результат для owner считается "успехом внедрения"?

### 2) Дашборд владельца
- [ ] Какие 5-7 метрик owner должен видеть регулярно?
- [ ] Какое управленческое действие принимается по каждой метрике?
- [ ] Откуда берутся данные по каждой метрике (1С/Bitrix/Excel/чаты/другое)?
- [ ] Как данные попадают к owner сейчас и как должны попадать после внедрения?
- [ ] Какой допустимый лаг данных (real-time / daily / weekly)?

### 3) Продажи и воронка
- [ ] Где конкретно теряются лиды по этапам?
- [ ] Какие точки контроля менеджеров нужны в первую очередь?
- [ ] Какие SLA/follow-up правила критичны для первой фазы?

### 4) Данные и системы
- [ ] Какие поля/события уже фиксируются стабильно?
- [ ] Какие данные есть, но им нельзя доверять?
- [ ] Какие интеграции/обмены сейчас самые проблемные?
- [ ] Что нельзя ломать в первой фазе (1С, Bitrix, регламенты)?

### 5) Роли и управление проектом
- [ ] Кто финальный decision maker?
- [ ] Кто владелец проекта внутри компании по дням?
- [ ] Кто отвечает за данные, кто за внедрение, кто за принятие изменений?

### 6) MVP, ресурсы и сроки
- [ ] Какой один процесс берем в MVP на 4-6 недель?
- [ ] Какой минимальный scope MVP без "большого взрыва"?
- [ ] Какие люди/часы/доступы нужны со стороны клиента?
- [ ] Какой горизонт: 6 месяцев (базовый контур), 12 месяцев (стабильная система)?

### 7) KPI и commit
- [ ] Какие 3 KPI должны улучшиться после первого этапа?
- [ ] Какая точка проверки результата через 2/4/6 недель?
- [ ] Фиксируем стратсессию сегодня: дата, время, участники, ответственные?
