# Эталон: план сборки sales-агента и личной RAG (Obsidian)

Статус: `ЭТАЛОН V1`  
Дата фиксации: `2026-03-04`

## 0) Цель

Собрать систему, которая:
1. Вытаскивает твой реальный опыт продаж из звонков, переписок и заметок.
2. Превращает его в повторяемые скиллы (`SKILL.md`).
3. Использует твою базу в Obsidian как RAG-контекст для подготовки сообщений, диагностик и follow-up.

## 1) Что у тебя уже есть (входные данные)

- Записи встреч.
- Записи командных обсуждений.
- Записи диагностик.
- Переписки с лидами.
- Истории чатов с ботами (подготовка/разбор).
- Чаты Telegram (источник лидов).
- Наброски логики, как ты ведешь лидов.
- Продукт: автоматизация на нейросетях + серверная агентная система.

Важно: полноценная воронка и конверсии пока не описаны. Это нормально для текущего этапа.

## 2) План работ (ускоренный, без жесткой привязки к 6 неделям)

### Шаг 1. Описать текущую воронку в черновике (1 вечер)

Сделай простую версию:
`Источник лида -> Первый контакт -> Диагностика -> Предложение -> Дожим -> Итог`

По каждому этапу зафиксируй:
- вход,
- твое действие,
- ожидаемый результат,
- типовые возражения,
- причина потери (если есть).

### Шаг 2. Собрать 20-30 кейсов из реальности (2-4 дня)

Приоритет источников:
1. Переписки с лидами.
2. Диагностики (текст/расшифровки).
3. Встречи и командные обсуждения.

На каждый кейс:
- стадия,
- что хотел клиент,
- что ты предложил,
- что сработало/не сработало,
- итог,
- причина потери или причина паузы.

### Шаг 3. Зафиксировать причины потерь (первый срез)

Минимальные категории:
- нет бюджета,
- не приоритет сейчас,
- не доверяет формату/технологии,
- не понял ценность,
- долгое согласование,
- пропал после N касаний,
- выбрал конкурента,
- "сырой" лид из канала.

### Шаг 4. Собрать первый пакет скиллов (MVP)

Сначала 3 штуки:
1. `lead-qualification-skill`
2. `diagnostic-summary-skill`
3. `followup-and-objection-skill`

Цель: чтобы агент помогал в реальных касаниях, а не в абстракции.

### Шаг 5. Поднять Obsidian RAG (MVP)

Структура папок:
- `00_Inbox`
- `10_Funnel`
- `20_Cases`
- `30_Objections`
- `40_Templates`
- `50_Skills`
- `90_Retrospective`

Теги:
- `#stage/...`
- `#source/telegram|meeting|diagnostic|chatbot`
- `#result/win|loss|pause`
- `#loss_reason/...`

### Шаг 6. Итерации каждую неделю

Ритм:
1. Разобрать 5-10 новых кейсов.
2. Обновить категории потерь.
3. Улучшить 1-2 скилла по реальным ошибкам.
4. Проверить, выросло ли качество ответов агента.

### Шаг 7. Client Intelligence Gate (до созвона)

Перед каждой диагностикой агент обязан собрать mini-dossier:
- кто человек (роль, зона ответственности, влияние на бюджет),
- что за компания (ниша, размер, стадия, релевантность под ICP),
- есть ли LPR или прямой доступ к LPR,
- есть ли бизнес-боль и горизонт решения 1-2 месяца,
- какой route: `A (диагностика)` / `B (nurture)` / `C (disqualify)`.

Минимальное правило входа в диагностику:
- LPR-score >= 2/3,
- ICP-fit >= 6/10,
- Pain/Urgency >= 2/3.

Если порог не пройден, звонок не назначается: только nurture-сценарий и повторная проверка через 2-6 недель.

## 3) Агент-сборщик данных (готовый промпт для отдельного чата)

Скопируй текст ниже в новый чат с агентом.

```text
Ты — мой Data Intake Agent для сборки персональной sales-RAG и sales-скиллов.

Контекст:
- Я продаю продукт: автоматизация с помощью нейросетей + серверная система на базе агентов.
- У меня уже есть: записи встреч, записи диагностик, переписки, чаты Telegram с лидами, чаты с ботами, заметки по логике продаж.
- У меня пока нет полной формализованной воронки и стабильной конверсии.

Твоя задача:
1) Вести меня пошагово и очень просто.
2) На каждом шаге просить один конкретный артефакт.
3) Если у меня нет артефакта, сразу предлагать альтернативу.
4) После каждого полученного артефакта коротко структурировать его в формате:
   - stage
   - source
   - summary
   - objections
   - next_action
   - result (win/loss/pause)
   - loss_reason (если есть)
5) Не переходить к следующему шагу, пока не обработан текущий.
6) Вести трекер прогресса: сколько кейсов собрано из целевых 30.

Правила диалога:
- Пиши коротко, без воды.
- Всегда говори "сейчас нужен только один следующий шаг".
- Если я пишу "нет", отвечай по шаблону:
  "Ок, тогда пришли [альтернатива-1]. Если нет, [альтернатива-2]."
- Если данные чувствительные, напомни обезличить имена и контакты.

Очередность запроса артефактов:
Шаг 1: 3-5 переписок с лидами (лучше разные исходы).
Шаг 2: 2-3 диагностики (текст/конспект/расшифровка).
Шаг 3: 2 записи встреч или их краткие заметки.
Шаг 4: 10 примеров возражений из чатов.
Шаг 5: 10 примеров follow-up сообщений (моих).
Шаг 6: Черновое описание текущей воронки (даже неполное).

Критерий завершения:
- Собрано минимум 20 кейсов,
- Есть первичная карта причин потерь,
- Есть материал для 3 sales-скиллов.

Начни прямо сейчас с Шага 1 и задай мне один конкретный запрос.
```

## 4) Что делать прямо сегодня (очень конкретно)

1. Создай отдельный чат и вставь туда промпт "Data Intake Agent".
2. Отправь в него первые 3 переписки (обезличенные).
3. Попроси у агента сразу выгрузить результат в формате markdown-заметок для Obsidian.
4. Сохрани эти заметки в `20_Cases`.
5. После 5 кейсов попроси агента сделать первый список `loss_reason`.

---

## 5) Definition of Done (MVP)

MVP готов, когда:
- Описана черновая воронка по твоему текущему продукту.
- Собрано >= 20 кейсов в едином формате.
- Зафиксированы причины потерь.
- Созданы 3 рабочих sales-скилла.
- Агент умеет готовить сообщения/диагностики с учетом твоего стиля и кейсов.
