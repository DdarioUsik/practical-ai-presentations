Use Case: Мониторинг производительности роботов с помощью n8n
Система отслеживания производительности роботов-уборщиков в реальном времени

Бизнес-задача
Компании с парком роботов для уборки столкнулась со сложностью мониторинга эффективности использования оборудования (25500+ задач ежемесячно). Ручной сбор данных о призводительности роботов занимал значительное время и не давал оперативной картины для оптимизации производственных процессов.
Реализованное решение
На базе n8n была построена система мониторинга, объединяющая сбор телеметрии с роботов, агрегацию метрик и визуализацию данных в единый автоматизированный поток отслеживания производительности.
Ключевые компоненты workflow

1. Scheduled триггер — запускает сбор данных по расписанию для регулярного обновления метрик.
2. HTTP Request модуль — получает телеметрию с API контроллеров роботов (статус, время работы, простои).
3. Модуль агрегации данных — рассчитывает коэффициенты производительности и KPI по каждому роботу и линии.
4. Интеграция с базой данных Airtable — сохраняет исторические данные для анализа трендов и построения отчётов.
5. Модуль расчёта аномалий — выявляет отклонения от нормальных показателей утилизации.
6. Dashboard обновление Airtable — передаёт актуальные данные в систему визуализации для мониторинга и отчетов.​
7. Alert-уведомления — отправляет оповещения по email при критическом снижении утилизации или сбоях, а также отправляет оформленные письма для клиентов .
Технические особенности
Вся логика реализована в n8n с использованием узлов Schedule Trigger, HTTP Request, Code, PostgreSQL и Email. Кастомный JavaScript выполняет расчёт коэффициентов утилизации по формулам OEE (Overall Equipment Effectiveness). Базой данных выступила система Airtable со встроенным дашбордом.
Оссобенностью являет дизайн системы выявления аномалий: настроены пороговые значения для каждого типа робота с учётом специфики производственного цикла. При снижении уровня перформанса (норма - average performance > 80%) более 3 дней подряд, система автоматически инициирует эскалацию внутренней команде.
Система обрабатывает данные с парка из 170+ роботов (~850 ежедневных задач) с интервалом опроса 24 часа, пакетами. Исторические данные хранятся для построения трендов и прогнозирования необходимости технического обслуживания.
