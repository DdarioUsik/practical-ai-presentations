Use Case: AI-модерация рекламных объявлений с помощью n8n
Автоматизированная система проверки крипто/финтех/гемблинг-рекламы

Бизнес-задача
Рекламная платформа обрабатывала тысячи объявлений в сфере криптовалют, финтеха и гемблинга, где ручная модерация приводила к ошибкам, разнобою в трактовке правил и задержкам запуска кампаний. Требовалась система с прозрачной трассировкой решений и автоматическим присвоением статусов Reject/Disallow/Risky/OK.
Реализованное решение
На базе n8n был построен многоуровневый конвейер модерации, объединяющий rule-engine, ML-модули анализа контента, интеграции с блокчейн-сканерами и KYC-провайдером в единый автоматизированный поток с полным audit trail.
Ключевые компоненты workflow
1. Webhook триггер — принимает рекламную кампанию при создании и оплате из внутренней системы.
2. Модуль валидации входа — проверяет заполненность полей, валидность URL и медиа, конфликты UTM.
3. OCR/NSFW анализатор — распознаёт текст на баннерах и выявляет запрещённый контент (военная символика, каннабис).
4. Классификатор типа объявления — определяет категорию: токен (launch/presale), NFT, казино, P2E, инвестиционный сайт.
5. Интеграция с Etherscan/DexScreener — проверяет смарт-контракты на mint/rebase, ликвидность и активность торгов.
6. KYC-интеграция (Sumsub) — получает статус верификации рекламодателя для расчёта пороговых правил.
7. Rule Engine + Decision Table — применяет матрицу решений по ликвидности, KYC-статусу и типу объявления.
8. Генератор вердиктов — формирует итоговое решение с метками (Investments, Gambling, NSFW) и шаблонным комментарием.
Технические особенности
Вся логика реализована в n8n с использованием узлов Webhook, HTTP Request (Etherscan, DexScreener, Sumsub API), Code для rule-engine и внешних ML-сервисов для OCR/NSFW. Правила хранятся в YAML/JSON с версионированием.
Критичным оказался дизайн пороговой матрицы ликвидности: токены с >50k получают статус OK, 1k-50k — Risky + Disallow для PooCoin без KYC, <500 без KYC — Reject. Система выявляет аномалии графика торгов (только покупки или только продажи).
Система обеспечивает до 90% автоматических решений без участия человека. Время обработки одного объявления — менее 60 секунд. Каждое решение логируется с полной трассировкой для аудита.
